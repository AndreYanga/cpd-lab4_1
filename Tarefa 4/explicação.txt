Tarefa 4: Transmissão vs. Envios Individuais
Este programa compara o desempenho do MPI_Bcast 
com o envio de uma grande matriz individualmente para cada nó.

Execução :

mpicc -o bcast_vs_send bcast_vs_send.c
mpirun -np 4 ./bcast_vs_send 1000000
mpirun -np 8 ./bcast_vs_send 1000000

Saída de exemplo (hipotética):
Array Size= 1000000 doubles, N Processes= 4
MPI_Bcast Time=     0.010000 sec
Individual Send Time=     0.030000 sec
Speedup (Send/Bcast)= 3.00
Array Size= 1000000 doubles, N Processes= 8
MPI_Bcast Time=     0.015000 sec
Individual Send Time=     0.060000 sec
Speedup (Send/Bcast)= 4.00

Espaço reservado para captura de tela : 
captura a saída do terminal para ambas as execuções.

Estatística descritiva (hipotética, substituir por dados reais):

4 Processos :
MPI_Bcast: Média = 0,010 s, Desvio Padrão = 0,001 s
Envio individual: Média = 0,030 s, Desvio padrão = 0,003 s
Aceleração: Média = 3,00, Desvio Padrão = 0,2
8 Processos :
MPI_Bcast: Média = 0,015 s, Desvio Padrão = 0,002 s
Envio individual: Média = 0,060 s, Desvio padrão = 0,005 s
Aceleração: Média = 4,00, Desvio Padrão = 0,3
Interpretação :

O MPI_Bcast é mais rápido porque usa comunicação coletiva otimizada 
(por exemplo, algoritmos baseados em árvore) em vez de envios sequenciais.
A aceleração aumenta com mais processos, pois os envios individuais 
são mal dimensionados (O(p) vs. O(log p) para transmissão).
